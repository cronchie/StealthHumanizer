# Architecture

StealthHumanizer is a full-stack web application built with Next.js that transforms AI-generated text into natural, human-sounding prose. It supports 15 AI model providers, features a multi-pass humanization pipeline, and includes a built-in AI detection engine.

## Module Descriptions

### `lib/providers.ts` вҖ” Provider Abstraction Layer

The heart of our multi-provider support. Implements a unified interface across 15 AI providers with vastly different APIs:

- **OpenAI-compatible adapters** вҖ” Groq, Together, OpenRouter, DeepInfra, Cerebras, Mistral all share the chat completions format
- **Native adapters** вҖ” Gemini (Google's generateContent API), Claude (Anthropic's Messages API), Cohere (chat API), HuggingFace (inference API)
- **Cloudflare Workers AI** вҖ” Edge inference with account-scoped URLs

Each provider normalizes to a single `generateWithProvider()` function that accepts `(provider, apiKey, systemPrompt, userPrompt, options)` and returns plain text.

### `lib/humanizer.ts` вҖ” Multi-Pass Humanization Engine

Orchestrates the full humanization pipeline:

1. **Chunking** вҖ” Long texts are split at sentence boundaries (max 2,500 words/chunk)
2. **Pass 1** вҖ” Full text rewrite using style-aware system prompts
3. **Detection** вҖ” Built-in detector scores each sentence
4. **Pass 2вҖ“3** (aggressive/ninja only) вҖ” Flagged sentences are re-humanized in batch
5. **Result assembly** вҖ” Sentence-level mapping with detection scores

### `lib/prompts.ts` вҖ” Style Engine & Prompt Engineering

Generates context-aware system prompts by composing:
- **Persona injection** вҖ” Different writing personas per rewrite level
- **Tone configuration** вҖ” 13 tone presets with personality traits, vocabulary, and writing patterns
- **Style guides** вҖ” 5 style presets (academic, casual, professional, creative, technical)
- **Level instructions** вҖ” 4 rewrite levels from light surgical edits to full ninja rewrites
- **Anti-pattern rules** вҖ” Explicit instructions to avoid AI-detectable patterns

### `lib/detector.ts` вҖ” AI Detection Engine

A heuristic-based detection system that analyzes text across 12 metrics:

| Metric | Weight | What It Measures |
|--------|--------|-----------------|
| Sentence Average | 25% | Combined per-sentence AI signals |
| Perplexity | 12% | Word/bigram predictability |
| Burstiness | 12% | Sentence length variation |
| Vocabulary Diversity | 8% | Unique word ratio |
| Sentence Length Variation | 8% | Range of sentence lengths |
| Transition Frequency | 8% | Overuse of transition words |
| Passive Voice Ratio | 5% | Passive construction frequency |
| AI Phrase Density | 7% | Known AI phrase matches |
| Sentence Start Diversity | 5% | Variety of sentence openers |
| Pronoun Usage | 5% | First/second person usage |
| Hedging Frequency | 3% | Hedging language density |
| Quantifier Overuse | 2% | Overuse of vague quantifiers |

### `lib/readability.ts` вҖ” Readability Metrics

Calculates standard readability scores: Flesch Reading Ease, Flesch-Kincaid Grade Level, Coleman-Liau Index, plus word/sentence/syllable counts and estimated reading time.

### `lib/storage.ts` вҖ” Client-Side Persistence

Manages localStorage for API keys, humanization history (max 50 entries), and theme preference. Includes text chunking, download utilities (TXT/DOCX), and word counting.

### `lib/types.ts` вҖ” TypeScript Type System

Comprehensive type definitions for providers, humanization options, detection results, readability scores, history entries, and UI state.

## Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| Framework | Next.js 16 (App Router) | Full-stack React framework |
| Language | TypeScript 6 | Type safety |
| Styling | Tailwind CSS 4 | Utility-first CSS |
| Icons | Lucide React | Icon library |
| AI Providers | 15 providers | Text generation via REST APIs |
| Storage | localStorage | Client-side persistence |
| Deployment | Vercel (recommended) | Zero-config Next.js hosting |

### Why This Stack?

- **Next.js** вҖ” Server-side API routes keep API keys off the client; App Router for modern React patterns
- **TypeScript** вҖ” Catches provider API mismatches at compile time; comprehensive type definitions
- **Tailwind CSS** вҖ” Rapid UI development; dark mode support built-in
- **No external AI SDKs** вҖ” Direct `fetch()` calls to provider APIs keep the bundle tiny and avoid dependency conflicts
- **localStorage** вҖ” Privacy-first; no server-side user data storage required

---

## Security Model

- API keys are stored in the browser's localStorage (encrypted provider keys sent over HTTPS)
- API routes act as a thin proxy вҖ” keys are forwarded to providers but never stored server-side
- No user accounts, no databases, no tracking вҖ” fully client-side architecture
- All provider communication happens over HTTPS/TLS
