# Contributing

For full contribution standards, see [CONTRIBUTING.md](https://github.com/rudra496/StealthHumanizer/blob/main/CONTRIBUTING.md).

## Quick Contribution Flow

1. Fork and create a topic branch from `main`.
2. Implement a focused change.
3. Run lint, integration smoke, and build.
4. Open a pull request using the repository template.

## Documentation Contributions

- Keep language concise and professional.
- Prefer relative links for internal docs.
- Update the changelog when documenting release-impacting changes.
